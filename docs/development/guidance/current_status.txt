# ACE Current Status - Quick Reference

**Date:** July 23, 2026

## ICLR 2027 Resubmission - In Progress

**Prior venue:** NeurIPS 2026 submission received critical reviews (4 of 5
reviewers flagged the same core issue: environment-query budget accounting
did not count lookahead probes, only executed interventions, making the
episode-count comparisons in Table 1 an apples-to-oranges comparison against
baselines that do not use lookahead). One reviewer (d6tT) also asked whether
DPO specifically was necessary, versus simpler preference-based updates.

**Decision:** Resubmit to ICLR 2027 rather than patch the NeurIPS version.
Reframe the paper's central claim around the LM-prior + lookahead-selection
mechanism (already the more defensible finding, per the anonymization and
ACE-w/o-DPO-tied-at-30-nodes results in the NeurIPS Section 5.2), with DPO
recast as one of several viable calibration layers rather than the headline
contribution. See the resubmission plan for full reasoning; do not edit the
plan file itself.

### Status by phase

**Phase 1 (code instrumentation): DONE**
- `--query counter`: every `env.generate()` call in `ExperimentExecutor` is
  now tagged (`executed`, `lookahead`, `observational`, `candidate_probe`)
  and logged per episode.
- `--lookahead_on_student`: scores the K candidates using the student's own
  forward pass instead of fresh oracle samples, so executed-interventions-only
  accounting is honest under this mode (zero additional oracle queries for
  lookahead).
- `--query_budget`: stops a run once cumulative environment samples hit a
  target, enabling total-query-matched comparisons across methods with very
  different per-decision oracle costs.
- `--policy_update {dpo, sft_best, ranking}`: SFT-on-best-candidate and a
  pairwise Bradley-Terry ranking loss (no reference-policy KL term) as
  DPO alternatives, reusing the existing candidate/lookahead machinery.
- Unit tests added for all four items (`tests/`).
- `baselines.py` audited for oracle-query multipliers: Max-Variance ~65x per
  decision (`n_candidates=64` probe generations), the 5-node Bayesian OED
  baseline in `run_reviewer_experiments.py` ~500x samples / 1001x calls per
  decision (`n_candidate_values=10` x `n_mc_samples=20` per one of 5 nodes),
  Random/Round-Robin/PPO at 1x (no candidate probing), ACE at 4-5x under
  env-based lookahead (K=4 candidates + 1 executed) and 1x under
  `--lookahead_on_student`. This audit is now Appendix "Budget Accounting:
  Full Multiplier Derivation" (`paper/iclr_ace_2027/appendix.tex`).
- Reward-function documentation bug fixed: the NeurIPS paper stated
  alpha=0.1 / gamma=0.05 for the node-importance and diversity terms; the
  code path actually exercised by every production run (`cov_bonus`
  default=60.0 with no separate alpha multiplier, `diversity_reward_weight`
  default=0.3) does not match. Corrected in the ICLR paper, appendix, and
  hyperparameter table.
- New `no_node_importance` ablation added (`--cov_bonus 0.0`) with CURC job
  scripts (`jobs/curc_submit_node_importance_ablation.sh`,
  `jobs/curc_node_importance_ablation_seed.sh`), 2 configs x 3 seeds.
